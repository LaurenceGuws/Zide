# Memory review (2026-01-15)

Scope: `src/` (active code). `old_editor_lib/` appears unused (reference only), so issues there are noted only if you later wire it in.

## Confirmed leak risks

1) `src/ui/renderer.zig:140-144`
- `Renderer.deinit` never unloaded a non-default font. If a custom font was loaded (via `loadFont` or `loadFontWithGlyphs`), its texture + glyph data leaked.
- Resolution: `Renderer.deinit` now unloads non-default fonts before teardown.

2) `src/ui/renderer.zig:197-233`
- `loadFontWithGlyphs` calls `c.LoadFontData` and `c.GenImageFontAtlas` (allocating `font_data` and `recs`). If `texture.id == 0` (or future early returns after `LoadFontData`), those allocations leaked because `UnloadFont` was never called.
- Resolution: added a cleanup path to unload the temporary font if `texture.id == 0`.

3) `src/ui/terminal_font.zig:46-55`
- `TerminalFont.init` returned early on failure without cleaning up prior resources.
  - Example: `FT_Init_FreeType` succeeds, then `FT_New_Face` fails -> `ft_library` leaks.
  - Example: `FT_New_Face` succeeds, then `FT_Set_Pixel_Sizes` fails -> both `ft_library` and `ft_face` leak.
- Resolution: added `errdefer` cleanup for FreeType + HarfBuzz init.

## Unbounded growth (not leaks, but memory can balloon)

4) `src/ui/terminal_font.zig:134-210`
- `TerminalFont.getGlyph` cached glyphs in a hashmap forever. Any unique glyph stayed resident until app exit.
- Resolution: added a bounded cache with FIFO eviction (`max_glyphs = 2048`).

5) `src/editor/buffer.zig` (piece table + undo)
- `add` buffer grows forever, and undo/redo stacks can accumulate lots of text over time.
- Resolution: added a cap on undo stack size (`max_undo_ops = 1000`).
- Remaining: piece-table `add` growth is still unbounded; consider compaction if long sessions are expected.

## Reference-only (not active unless you wire it in)

- `old_editor_lib/zig/src/syntax.zig:134-205`
  - `highlightRange` allocates output; callers must call `zig_syntax_free_highlights` or it leaks.
  - If you hook this library in, ensure the caller frees the buffer.

## Notes

- I did not see references from `src/` or build files to `old_editor_lib`, so it appears unused for now.
