### Recent Changes (High-Level)
- The high-refresh rain investigation removed most renderer-side force-full and stale invalidation escape hatches.
- Full-screen `ascii-rain` is now close to stable, so the stronger remaining work is structural rather than incident-driven.
- `src/app` dumping-ground cleanup materially moved forward: terminal, search, editor, and tabs now have focused app subtrees instead of a flat root namespace.
- `src/editor/editor.zig` large-file cleanup is active: search worker/highlighter logic now lives under `src/editor/search_highlight.zig`, selection/multi-caret/undo-selection state under `src/editor/selection_state.zig`, cursor navigation/selection-extension behavior under `src/editor/navigation.zig`, and text mutation flows under `src/editor/edit_ops.zig`, with `editor.zig` shrinking toward a real facade over editor state ownership.
- `src/editor/syntax.zig` large-file cleanup is effectively landed: syntax query loading/capture mapping/query-cache ownership now live under `src/editor/syntax_queries.zig`, runtime highlighter/injection execution under `src/editor/syntax_runtime.zig`, predicate/directive evaluation plus node-text/regex helpers under `src/editor/syntax_predicates.zig`, and token overlap post-processing under `src/editor/syntax_tokens.zig`, leaving `syntax.zig` as a small root facade instead of another mixed-responsibility file.
- `src/ui/widgets/terminal_widget_draw.zig` large-file cleanup is now active: the grid/text rendering owner has moved into `src/ui/widgets/terminal_widget_draw_grid.zig`, which now owns row background drawing, shaping runs, underline emission, sprite/box glyph rendering, and glyph-cache submission. The root draw file now leans more toward texture-update orchestration, overlays, cursor, selection, and scrollbar chrome.
- `src/ui/widgets/terminal_widget_draw.zig` large-file cleanup has moved forward again: overlay/chrome ownership now also lives in `src/ui/widgets/terminal_widget_draw_overlay.zig`, covering selection overlays, hover underline, cursor drawing, IME composition preview, scrollbar chrome, and the scrollback badge. The root file is now primarily texture-update orchestration and the remaining kitty/presentation glue.
- `src/ui/widgets/terminal_widget_draw.zig` large-file cleanup is effectively landed: grid/text rendering now lives in `src/ui/widgets/terminal_widget_draw_grid.zig`, overlay/chrome in `src/ui/widgets/terminal_widget_draw_overlay.zig`, and viewport-shift/texture-update/partial redraw planning in `src/ui/widgets/terminal_widget_draw_texture.zig`. The root file is now coordinator-sized, with only top-level kitty/presentation/perf glue left in place.
- `src/ui/widgets/editor_widget_draw.zig` large-file cleanup is now active: draw-list primitives, selection/search overlays, extra caret helpers, selection-state hashing, and scrollbar rendering now live in `src/ui/widgets/editor_widget_draw_overlay.zig`, leaving the root file more focused on text/layout/highlight assembly plus cached/live draw orchestration.
- `src/ui/widgets/editor_widget_draw.zig` cleanup moved forward again: byte-range mapping, selection-text background splitting, token color selection, and highlighted segment rendering now live in `src/ui/widgets/editor_widget_draw_text.zig`. The root file is now largely the draw entrypoint/coordinator plus precompute/hash/fallback helpers.
- `src/ui/widgets/editor_widget_draw.zig` cleanup is now effectively landed: draw-list/overlay/scrollbar helpers live in `src/ui/widgets/editor_widget_draw_overlay.zig`, text/highlight segment assembly in `src/ui/widgets/editor_widget_draw_text.zig`, and fallback/precompute/hash planning in `src/ui/widgets/editor_widget_draw_cache.zig`. The root file is now coordinator-sized instead of another large mixed-responsibility draw file.
- `src/ui/renderer.zig` large-file cleanup is now active: SDL/input event polling, event dispatch, and frame queue compaction now live in `src/ui/renderer/input_runtime.zig`, leaving the root renderer file thinner around top-level renderer orchestration instead of also owning the full input runtime loop inline.
- `src/ui/renderer.zig` cleanup moved forward again: font-feature config plus UI-scale/zoom runtime now live in `src/ui/renderer/font_runtime.zig`, leaving the root renderer file thinner around rendering entrypoints instead of also carrying font runtime policy inline.
- `src/ui/renderer.zig` cleanup moved forward again: text/glyph rendering, shaping, and terminal cell draw paths now live in `src/ui/renderer/text_runtime.zig`, leaving the root renderer file thinner around draw coordination and backend wrappers instead of also owning the heaviest text rendering internals inline.
- `src/ui/renderer.zig` cleanup is now effectively landed: SDL/input event polling lives in `src/ui/renderer/input_runtime.zig`, font-feature config plus UI-scale/zoom runtime in `src/ui/renderer/font_runtime.zig`, and text/glyph rendering in `src/ui/renderer/text_runtime.zig`. The root file is now coordinator-sized rather than another large mixed-responsibility renderer blob.
- `src/ui/terminal_font.zig` cleanup is now active: atlas allocation/upload/compaction now lives in `src/ui/font/atlas.zig`, establishing `src/ui/font/` as the focused font subsystem root and starting the split of the remaining font selection/shaping/fallback owners out of the old giant file.
- `src/ui/terminal_font.zig` cleanup moved forward again: preferred/fallback face selection policy now lives in `src/ui/font/fallback.zig`, reducing the root font file around shaping and system font discovery instead of also carrying the full fallback preference matrix inline.
- `src/ui/terminal_font.zig` cleanup moved forward again: special-glyph sprite generation, powerline outline rasterization, and sprite-cache population now live in `src/ui/font/special_glyphs.zig`, reducing the root font file around regular glyph shaping/draw flow and system font discovery instead of also carrying the full special-glyph sprite path inline.
- `src/ui/terminal_font.zig` cleanup moved forward again: regular glyph shaping, grapheme shaping, glyph-advance lookup, and draw placement now live in `src/ui/font/shaping.zig`, reducing the root font file around face selection and system font discovery instead of also carrying the full HarfBuzz draw path inline.
- `TerminalSession` surface reduction is largely landed: borrowed title/cwd/scrollback/selection/input/interaction/rendering/protocol/config/runtime seams were cut back, terminal text export now lives in backend code instead of the widget, FFI/workspace/open-path callers use backend-owned metadata/export contracts instead of separate raw getter calls, split title/cwd helpers, or a dedicated child-exit getter, metadata/hyperlink/clipboard/close-confirm query logic now lives under `src/terminal/core/session_queries.zig`, generation/backlog/presentation-feedback bookkeeping under `src/terminal/core/session_publication.zig`, scrollback/text-export/scroll-offset content helpers under `src/terminal/core/session_content.zig`, selection passthrough helpers under `src/terminal/core/session_selection.zig`, key/char/text/mouse/focus/color-scheme send paths under `src/terminal/core/session_input.zig`, input-state query plus OSC 5522 paste helpers under `src/terminal/core/session_interaction.zig`, snapshot/render-cache/sync-updates damage-retirement helpers under `src/terminal/core/session_rendering.zig`, parser/control/screen-edit/alt-screen/DECRQSS protocol helpers under `src/terminal/core/session_protocol.zig`, palette/theme/column-mode/cell-size config helpers under `src/terminal/core/session_config.zig`, and PTY startup/poll/child-exit/write-lock/resize runtime helpers under `src/terminal/core/session_runtime.zig` instead of sitting inline in `terminal_session.zig`.
- Workspace/session boundary tightening is now active: app clipboard shortcuts no longer resolve sessions through workspace, the dead app-side active-session resolver was deleted, workspace tab creation now has an explicit `createTabWithSession(...)` contract, the remaining raw workspace session helpers (`sessionAt`, `activeSession`) were pushed back to internal-only helpers, terminal tab-bar sync now consumes one workspace-owned `copyTabSyncState(...)` contract, and window-close confirm routing now uses `firstConfirmCloseTab(...)` instead of app-side tab scanning.
- Runtime scheduling ownership cleanup is now effectively landed: workspace owns active-frame observation (`activeFrameState()`) and per-frame poll publication (`PollFrameResult`), runtime owns poll profile selection and sleep cadence through explicit runtime profiles, and terminal idle/backoff bookkeeping lives inside `TerminalFramePacingState` instead of generic `AppState` fields.
- Widget-input reduction also moved forward: terminal scrollbar drag state is now widget-owned instead of living in generic app state/glue, and click/drag/scroll gestures now lean on backend-owned selection/scroll helpers instead of duplicating row/word/anchor policy in `terminal_widget_input`.
- Widget/backend presentation ownership also moved forward: terminal widget draw now goes through backend-owned presentation capture/finish methods instead of assembling published-cache copy timing and feedback handoff inline in the widget.
- Clipboard/paste ownership also moved forward: system and middle-click selection paste fallback policy now lives in backend code instead of widget code, including scrollback reset for system paste, OSC 5522 attempt, bracketed paste framing, and plain-text fallback behavior.
- Scrollback-view collapse policy also moved forward: widget input no longer owns the actual “snap back to live bottom on input activity” rule; that decision now goes through backend scrollback helpers.
- Terminal widget input ownership also moved forward: the keyboard/text dispatch path, the locked pointer/selection/scrollback path, and the mouse-reporting path now live in dedicated UI helper modules instead of staying embedded inside `terminal_widget_input.zig`, making the widget input file a top-level coordinator over helper modules rather than another large mixed-responsibility block.
- Investigation residue cleanup is active: probe-grade scroll/control trace logging was removed from core hot paths, protocol trace noise now defaults to debug level, high-frequency input/scrollback send-path logs were demoted out of default-info output, raw CSI-byte tracing now defaults to debug, and duplicate stdout lifecycle logs were removed.
- Current high-risk architectural seams are:
  - `TerminalSession` is still a large multi-domain owner (PTY/parser/screens/history/render publication/UI-facing APIs).
  - terminal-originated PTY writes and the main session mutation/publication locking cleanup are now in better shape, so the next structural hotspot is the oversized `TerminalSession` surface and the borrowed app/UI query seams hanging off it.
  - redraw lifecycle ownership is significantly cleaner: published cache capture and post-draw completion are now behind backend APIs, although `view_cache`, widget wrapper code, and frame runtime still participate in the presentation pipeline.
  - scheduler/poll state is still split across app runtime helpers and `TerminalWorkspace`, but concrete workspace poll budgets now live behind the workspace contract instead of the app hook.
- protocol/parser seam is still only partially typed; the VT parser callback contract in `parser.zig`, top-level parser dispatch in `parser_hooks`, DCS/APC/OSC/CSI parser dispatch, OSC title/hyperlink/cwd/clipboard/semantic/palette/kitty-clipboard boundaries, and kitty paste-event emission/internal helper paths now run through explicit session facades/context structs (including OSC 5522 write/reply flow through an explicit `WriterFacade`). In parser text-write handling, codepoint/ascii execution in `parser_hooks` now runs through an explicit text-write context instead of reaching through raw session shape. OSC top-level dispatch now routes through a typed composite facade over the split OSC submodules instead of another opaque trampoline layer, and the helper modules for title, hyperlink, cwd normalization, OSC 52 clipboard, semantic prompt/user vars, and palette/dynamic colors now use direct typed state holders wherever the logic already owns that state instead of layering wide per-call opaque callback facades on top. In CSI, reply emission now runs through `CsiWriter`, query/reply data through `QueryContext`, screen cursor/grid reply reads through `ScreenQueryContext`, simple cursor/edit/scroll execution through `SimpleCsiContext`, `SM`/`RM` mode toggles through `ModeMutationContext`, `DECRQM` snapshot capture through `ModeQueryContext`, the remaining simple special-case control branches through `SpecialCsiContext`, and reply/control lock-dispatch orchestration through `ReplyCsiContext`; `DECSTR` now runs through an explicit reset context and `applySgr` through an explicit SGR context. `dcs_apc` also no longer bounces kitty APC payloads back through a public `TerminalSession.parseKittyGraphics(...)` wrapper. Remaining implicit seams are narrower but still exist in selected CSI policy corners, protocol entry wrappers, and some widget/backend boundaries.
- Kitty graphics is no longer a single mixed-responsibility blob. Shared kitty state/types, payload transport/build, placement-graph mutation/scroll/placement helpers, byte-store lifecycle, and protocol/delete/reply orchestration now live in dedicated `src/terminal/kitty/common.zig`, `src/terminal/kitty/transport.zig`, `src/terminal/kitty/placement_ops.zig`, `src/terminal/kitty/storage_ops.zig`, and `src/terminal/kitty/protocol_ops.zig` modules. `graphics.zig` is now mostly a thin facade/re-export surface plus tests.
  - widget input/draw still contain backend-policy behavior rather than being thin presentation/orchestration layers.
  - selection semantics are cleaner than before because row-content/word/range rules now live in backend model/core code, but the widget still owns substantial interaction orchestration and presentation policy.

### Known Risk (High-Level)
- A broad refactor done in the wrong order will re-entangle correctness with redraw/scheduler changes and make regressions hard to localize.
- The main remaining risk is boundary drift: session/runtime/widget/protocol code still share responsibilities that should be isolated.
- Kitty graphics, sync/presentation ownership, and input snapshot publication remain sensitive correctness surfaces.
